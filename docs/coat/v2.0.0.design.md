## 1. 구조 개요

두 기능은 독립적이지만 같은 데이터 레이어(`.coat/state/memory.json`)를 공유한다.

```
[B. 멀티프로젝트]
  npm run dashboard -- --projects /path/a,/path/b
    └─ server.js: --projects 파싱 → PROJECT_LIST 배열
    └─ GET /api/projects → 전체 프로젝트 상태 목록
    └─ GET /api/state?project=0 → 특정 프로젝트 상태
    └─ dashboard.html: 사이드바 + 프로젝트 전환 로직

[E. VSCode 상태바]
  vscode-extension/
    ├─ package.json       ← VSCode extension manifest
    ├─ extension.ts       ← activate/deactivate, StatusBarItem, FileWatcher
    └─ tsconfig.json
  동작:
    activate() → .coat/state/memory.json 존재 확인
    존재 시 → StatusBarItem 생성 + fs.watch 등록
    변경 시 → memory.json 읽기 → 상태바 텍스트 업데이트
    클릭 시 → WebviewPanel 열기 (체크리스트 표시)
    .coat/ 없으면 → StatusBarItem.hide()
```

## 3. 데이터 흐름

```
[B] 멀티프로젝트 흐름
  서버 시작
    → --projects /a,/b,/c 파싱 → PROJECT_LIST = ['/a', '/b', '/c']
    → --project /a 이면 → PROJECT_LIST = ['/a'] (기존 단일 모드)

  GET /api/projects
    → 각 프로젝트 .coat/state/memory.json 읽기
    → 요약 배열 반환:
      [
        { index: 0, name: "프로젝트A", feature: "소셜로그인", phase: "loop", round: 8 },
        { index: 1, name: "프로젝트B", feature: "v1.3.0", phase: "completed", round: 3 }
      ]

  GET /api/state?project=0
    → PROJECT_LIST[0] 기준으로 기존 getState() 호출
    → project 파라미터 없으면 PROJECT_LIST[0] 기본값

  대시보드 초기화
    → /api/projects 호출 → 사이드바 렌더
    → activeProject = 0 → /api/state?project=0 호출 → 메인 패널 렌더
    → SSE /api/events?project=0 연결

  사이드바 클릭 (project=1)
    → activeProject = 1
    → 기존 SSE 닫기 → 새 SSE /api/events?project=1 연결
    → /api/state?project=1 호출 → 전체 리렌더

[E] VSCode 상태바 흐름
  extension activate
    → workspaceFolders 확인
    → .coat/state/memory.json 존재 확인
      없으면 → statusBarItem.hide(), fs.watch('.coat/') 대기
      있으면 → updateStatusBar() + fs.watch 등록

  updateStatusBar()
    → memory.json 읽기
    → phase = "loop"  → "🧥 {feature} · LOOP R{round} | 기능{N}%"
    → phase = "plan"  → "🧥 {feature} · PLAN"
    → phase = "completed" → "🧥 완료: {feature}"
    → statusBarItem.show()

  클릭 이벤트
    → WebviewPanel 생성
    → memory.json의 checklist 읽어서 HTML 렌더
    → Match Rate 게이지 포함

  .coat/ 디렉토리 생성 감지
    → statusBarItem 재활성화
```
