## 1. 구조 개요

세 가지 독립 개선이다. R-01은 SKILL.md의 next 로직을 "안내 → 확인 후 자동 실행"으로 격상하고, R-02는 server.js에 SSE 엔드포인트를 추가하고 dashboard.html의 폴링을 EventSource로 교체하며, R-03은 wrap 로직에 README.md 버전 히스토리 행 추가를 넣는다.

```
R-01: /coat next
        현재: "DESIGN으로 넘어가세요: /coat design {기능명}"
        변경: "다음 단계: DESIGN. 바로 시작할까요? [예] [아니오]"
              [예] 선택 시 → /coat design {기능명} 자동 실행

R-02: server.js
        GET /api/events  → SSE 스트림
        fs.watch(.coat/state/) 로 파일 변경 감지
        변경 시 → "data: update\n\n" 전송
      dashboard.html
        new EventSource('/api/events') 로 교체
        onmessage → poll() 호출
        setInterval(poll) 제거
        연결 끊김 시 3초 후 재연결 (자동 복구)

R-03: /coat wrap 로직
        README.md 파일에서 버전 히스토리 테이블 찾기
        마지막 행 다음에 새 행 추가:
        | v{버전} | {YYYY-MM-DD} | {기능명} |
```

## 3. 데이터 흐름

```
[R-01] /coat next 실행
  → memory.json 읽기 → phase 확인
  → 다음 단계 명령 제시 + "바로 시작할까요? [예] [아니오]"
  → [예]: 해당 단계 명령 즉시 실행
  → [아니오]: 명령어만 출력하고 종료

[R-02] SSE 흐름
  서버:
    클라이언트 연결 → res.writeHead(200, SSE 헤더)
    fs.watch('.coat/state/') 등록
    파일 변경 → res.write('data: update\n\n')
    클라이언트 연결 종료 → fs.watch 해제

  클라이언트:
    페이지 로드 → new EventSource('/api/events')
    onmessage → poll() (API /api/state 호출 후 렌더)
    onerror → 3초 후 EventSource 재생성
    setInterval(poll, 2000) 제거
    초기 1회 poll() 유지 (첫 화면 즉시 로딩)

[R-03] /coat wrap 실행
  → README.md 읽기
  → "| v" 로 시작하는 버전 히스토리 테이블 마지막 행 탐색
  → 새 행 삽입: | v{기능명/버전} | {날짜} | {설명} |
  → README.md 저장
  → "README.md 버전 히스토리 업데이트됨"
```
