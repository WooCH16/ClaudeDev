## 1. 구조 개요

세 가지 독립적인 개선이 병행된다. R-01은 SKILL.md의 mini wrap 로직에 `_INDEX.md` 갱신을 추가하고, R-02는 dashboard.html에 CSS 변수 교체 방식의 테마 토글을 추가하며, R-03은 wrap 로직과 history.json 스키마에 `round` 필드를 추가한다.

```
R-01: /coat mini wrap
        └─ .coat/audit/p1-records/{이슈명}-{날짜}.md 저장
        └─ _INDEX.md 읽기 → 항목 추가 → 저장 (없으면 신규 생성)

R-02: dashboard.html
        └─ <button id="themeToggle"> 헤더에 추가
        └─ localStorage["coat-theme"] = "light" | "dark"
        └─ CSS :root 변수를 [data-theme="light"] 로 오버라이드

R-03: /coat wrap
        └─ memory.json의 round 값 읽기
        └─ history.json 항목에 "round": N 추가
     history.json 스키마
        └─ { feature, completedAt, round, matchRate, backlog }
```

## 2. 컴포넌트 목록

| 컴포넌트 | 역할 | 변경 파일 |
|---------|------|---------|
| mini wrap 로직 | p1-record 저장 후 _INDEX.md 갱신 | SKILL.md |
| _INDEX.md 포맷 | 전체 목록 + 해결률 표 | SKILL.md (신규 정의) |
| 테마 토글 버튼 | 헤더 우측, GitHub 배지 옆 | dashboard.html |
| CSS 라이트 테마 | `[data-theme="light"]` 변수 오버라이드 | dashboard.html |
| themeToggle JS | 클릭 시 data-theme 전환 + localStorage 저장 | dashboard.html |
| wrap round 기록 | history.json 항목에 round 필드 추가 | SKILL.md |
| history.json 스키마 | round 필드 추가 | schemas/history.json |

## 3. 데이터 흐름

```
[R-01] mini wrap 실행
  → p1-record 파일 저장
  → _INDEX.md 존재 확인
    존재: 파일 읽기 → 항목 추가 → 해결률 재계산 → 저장
    없음: 헤더 + 항목 1개로 신규 생성
  → "INDEX 갱신됨: .coat/audit/p1-records/_INDEX.md"

[R-02] 페이지 로드
  → localStorage["coat-theme"] 읽기
  → document.documentElement.dataset.theme = 저장값 | "dark"
  버튼 클릭
  → dark ↔ light 전환
  → localStorage 저장
  → 버튼 아이콘 갱신 (🌙 / ☀️)

[R-03] wrap 실행
  → memory.json.round 읽기
  → history.json 항목 생성: { feature, completedAt, round, matchRate, backlog }
  → 저장
```
